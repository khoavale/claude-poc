## Description
Build the `HeroBanner` React component: a full-width, full-bleed image section with a dark overlay, a bottom-left-positioned headline and CTA button. Content (headline, image, CTA label, CTA href) is CMS-driven via the `useHeroPromotion` hook. Performance is critical: the hero image must use `fetchpriority="high"`, a `srcset` with three size variants, and must not be lazy-loaded.

## Acceptance Criteria
- [ ] `src/components/HeroBanner.tsx` created as a named export
- [ ] Full-width (`100vw`), no max-width constraint; height responsive: `560px` (lg), `420px` (md), `320px` (sm)
- [ ] Top margin `56px` to clear the fixed navbar
- [ ] Background image: `object-fit: cover`, `object-position: center`; dark overlay `rgba(0,0,0,0.18)` applied via an absolutely positioned div
- [ ] Headline renders CMS `headline` field; font `64px/68px`, `font-black`, `uppercase`, `color-brand-white`; max-width `480px`; responsive: `48px/52px` at md, `36px/40px` at sm
- [ ] CTA button labelled with CMS `ctaLabel`; routes to CMS `ctaHref`; hero variant (white bg / black text); height `48px`, padding `0 24px`, min-width `200px`; full-width on mobile
- [ ] CTA button hover state: `color-neutral-100` background per UX spec (not primary-hover which is dark)
- [ ] Hero `<img>` has `fetchpriority="high"`, is NOT `loading="lazy"`, has `srcset` with at minimum three size variants (640w, 1024w, 1440w) ‚ÄĒ actual URLs built from `imageUrl` + width suffix convention
- [ ] `<img alt="">` text comes from CMS `imageAlt` field ‚ÄĒ never hardcoded
- [ ] Section wrapped in `<section aria-label="Promotional hero">`
- [ ] Fallback content (static placeholder headline + default CTA) renders when `useHeroPromotion` returns an error
- [ ] Loading state renders a grey skeleton at the correct height
- [ ] `prefers-reduced-motion`: CTA hover transition duration is 0ms
- [ ] LCP acceptance test: Lighthouse on a simulated 4G reports LCP ‚Č§ 2.5s (verified by QA, not Dev, but Dev must implement the performance prerequisites)

## Technical Notes
- `fetchpriority="high"` is an HTML attribute; in React: `fetchPriority="high"` (camelCase)
- `srcset` pattern: given `imageUrl = "https://cdn.example.com/hero.webp"`, generate `"https://cdn.example.com/hero-640w.webp 640w, ..."` ‚ÄĒ confirm URL pattern with the CMS/CDN convention; use a `buildHeroSrcSet(baseUrl: string): string` utility function in `src/utils/image.ts`
- The overlay div must sit between the `<img>` and the text container in the DOM; use `absolute inset-0` positioning
- Bottom-left text container: `absolute bottom-[80px] left-[48px]` on desktop; responsive padding from UX spec
- Text and button stack vertically with `gap: 24px`
