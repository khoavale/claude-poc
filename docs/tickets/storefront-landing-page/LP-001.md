# LP-001 — Design tokens: Tailwind config and CSS custom properties

## Type
[x] Chore  [ ] Feature  [ ] Bug  [ ] Migration

## Linked spec
docs/specs/storefront-landing-page.md — all sections (visual foundation required by every UI ticket)

## Linked UX spec (if UI work)
docs/ux/storefront-landing-page.md — Colours, Typography, and Spacing tables across all sections

## Description
Establish the project-level Tailwind configuration and CSS custom properties that underpin every UI component on the storefront landing page. This covers all colour tokens (brand, neutral, hero, sport, footer, wishlist, button states), spacing tokens (nav-height, hero-height variants, product-card dimensions), typography tokens (text-hero, nav-logo-font-size, letter-spacing-wide/wider), shadow tokens (shadow-nav, shadow-sm, shadow-md), and transition tokens (transition-fast, transition-base, transition-slow). Nothing visual can be built correctly without this foundation in place.

## Technical Notes
- Extend `theme.extend` — never override `theme` directly, to preserve Tailwind defaults
- Use CSS custom properties (`--color-brand-white: #FFFFFF`) as the backing layer so tokens are accessible to non-Tailwind code (e.g., SVG stroke colour via `currentColor` or inline for SVGs)
- Breakpoints must be confirmed or set: `sm: 320px`, `md: 768px`, `lg: 1024px` — align with UX responsive spec
- The `prefers-reduced-motion` rule can be added as a Tailwind plugin or a global CSS class applied at the root
