# LP-004 — React Query hooks for storefront landing page API

[x] Feature  [ ] Chore  [ ] Bug  [ ] Migration

## Linked spec
docs/specs/storefront-landing-page.md — US-02, US-03, US-05, US-06

## Linked UX spec (if UI work)
N/A — this ticket is data-layer only; no UI components are built here

## Description
Implement the React Query hooks that fetch the four storefront landing page content resources (hero promotion, new arrivals products, sport categories, membership content). These hooks are the sole data-fetching interface the UI components (LP-005 onwards) will consume. They wrap the Axios/fetch API client functions and use React Query's `useQuery` pattern as specified in `agents/sa/agent.md`.

## Acceptance Criteria
- [ ] `src/api/storefront.ts` created — plain async functions: `fetchHeroPromotion`, `fetchNewArrivals`, `fetchSportCategories`, `fetchMembershipContent` — each returns the corresponding type from LP-002
- [ ] `src/api/hooks/useStorefront.ts` created with four named exports:
  - `useHeroPromotion()` — `useQuery({ queryKey: ['storefront', 'hero'], queryFn: fetchHeroPromotion })`
  - `useNewArrivals()` — `useQuery({ queryKey: ['storefront', 'new-arrivals'], queryFn: fetchNewArrivals })`
  - `useSportCategories()` — `useQuery({ queryKey: ['storefront', 'sport-categories'], queryFn: fetchSportCategories })`
  - `useMembershipContent()` — `useQuery({ queryKey: ['storefront', 'membership'], queryFn: fetchMembershipContent })`
- [ ] All hooks return `{ data, isLoading, isError }` — callers handle loading and error states
- [ ] `staleTime` set to `5 * 60 * 1000` (5 minutes) for all storefront queries — content changes infrequently
- [ ] No `any` types; all return types explicitly annotated using interfaces from `src/types/storefront.ts`
- [ ] Unit tests (Vitest + MSW or similar) cover: successful fetch returns typed data, network error sets `isError`

## Technical Notes
- Follow the `useProducts` / `useCreateProduct` pattern exactly from `agents/sa/agent.md`
- API base URL read from `import.meta.env.VITE_API_BASE_URL` — never hardcoded
- No barrel re-exports that create circular dependencies; `useStorefront.ts` imports from `@/api/storefront` not from `@/api`
- These hooks do not handle wishlist state — that is a Zustand concern (LP-005)
