## Description
Implement four read-only Express API endpoints that serve the storefront landing page content. All content is treated as CMS-driven per the UX ambiguity resolutions. For the MVP the "CMS" is seeded data in SQLite; the response shape must match the TypeScript interfaces defined in LP-002. No write endpoints are in scope for this ticket.

## Acceptance Criteria
- [ ] `GET /api/storefront/hero` returns `HeroPromotion` JSON: `{ headline, imageUrl, imageAlt, ctaLabel, ctaHref }`
- [ ] `GET /api/storefront/new-arrivals` returns an array of up to 3 `StorefrontProduct` objects ordered by `arrivedAt` descending; query parameter `?limit=3` is accepted (default 3, max 3 for this endpoint)
- [ ] `GET /api/storefront/sport-categories` returns an ordered array of `SportCategory` objects (max 3 per UX A-05); order is by a `displayOrder` column
- [ ] `GET /api/storefront/membership` returns `MembershipContent` JSON: `{ heading, body, imageUrl }`
- [ ] All endpoints return HTTP 200 on success; HTTP 500 with `{ message: 'Internal server error' }` on unexpected errors
- [ ] Input validated with Zod where query params are accepted (limit param on new-arrivals)
- [ ] Routes are mounted under `/api/storefront` in the main Express app
- [ ] Each endpoint uses Prisma `select` to fetch only needed columns ‚Äî no `SELECT *`
- [ ] Unit tests (Vitest or Jest) cover: successful response shape, empty-array edge case for new-arrivals and sport-categories

## Technical Notes
- Create `src/server/routes/storefront.ts` ‚Äî follow the Router pattern in `agents/sa/agent.md`
- Seed data: create a `prisma/seed.ts` or add to the existing seed file with representative records (hero, 3 products, 3 sport categories, membership copy)
- The `new-arrivals` endpoint always slices to 3; the business logic (ordering + slicing) happens in the DB query, not in application code
- Prisma models needed: `HeroPromotion`, `StorefrontProduct` (or reuse `Product` with a flag), `SportCategory`, `MembershipContent` ‚Äî these models are created in LP-003a (see dependency note)
- If a `products` table already exists from another feature, add an `arrivedAt` column via a new migration rather than creating a duplicate table

- Admin content management endpoints
- Image upload or CDN integration ‚Äî `imageUrl` is a plain string; actual asset hosting is out of scope
