## Description
Implement the Zustand client-state stores for cart items and wishlist items. These stores provide the item counts consumed by the Navbar badge and the toggle state consumed by the product card wishlist heart. They are client-only state ‚Äî no persistence to the API in this ticket (the wishlist API mutation is LP-006). The stores must expose counts so the Navbar badge always reflects the current in-memory state.

## Acceptance Criteria
- [ ] `src/store/useCartStore.ts` created with: `items: CartItem[]`, `add(item: CartItem): void`, `remove(productId: string): void`, `clear(): void`, `count: number` (derived selector or computed)
- [ ] `src/store/useWishlistStore.ts` created with: `items: WishlistItem[]`, `toggle(productId: string): void` (adds if not present, removes if present), `isWishlisted(productId: string): boolean`, `count: number`
- [ ] `CartItem` and `WishlistItem` types imported from `@/types/storefront` (defined in LP-002) ‚Äî no inline type redeclarations
- [ ] `useCartStore` and `useWishlistStore` are named exports from their respective files
- [ ] Both stores follow the Zustand pattern from `agents/sa/agent.md` (`create<State>((set, get) => ({...}))`)
- [ ] No `any` types
- [ ] `useWishlistStore.toggle` is idempotent: toggling the same product twice returns to the original state
- [ ] Unit tests cover: initial state is empty, `toggle` adds item, `toggle` same item removes it, `count` is accurate

## Technical Notes
- `count` can be a simple selector: `(state) => state.items.length` ‚Äî do not store it as a separate field to avoid stale state
- `isWishlisted` uses `get()` inside the store or is a plain selector used by components ‚Äî both are acceptable; prefer the selector pattern used by the component for clarity
- These stores intentionally do not persist to `localStorage` at this stage; persistence is a post-launch enhancement
- The wishlist `toggle` in this ticket is optimistic / client-only; the actual API call (to persist wishlist server-side) is LP-006

## Out of Scope

- Cart persistence or checkout flow
- Authentication-gated wishlist (the auth check before saving is handled in LP-006 and the UI component in LP-007)
