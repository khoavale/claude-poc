## Description
Build the `MembershipSection` component: a two-column flex section with CMS-driven heading and body copy on the left, and a membership photograph on the right. The section renders "JOIN US" and "SIGN IN" CTAs for unauthenticated users, and a "GO TO MY ACCOUNT →" link for authenticated users (UX A-07). Content is CMS-driven via `useMembershipContent` (UX A-06).

## Acceptance Criteria
- [ ] `src/components/MembershipSection.tsx` created as a named export
- [ ] Section layout: flex row, `align-items: center`, `gap: 48px`; section padding `64px` top/bottom, `40px` horizontal
- [ ] Left column: `flex: 1`, max-width `400px`; contains heading, body copy, button row
- [ ] Right column: `flex: 0 0 auto`, width `420px`, height `400px`; membership photograph `object-fit: cover`; fades in from `opacity: 0` to `1` on mount with `transition-slow`; `loading="lazy"`
- [ ] Section heading: `text-2xl`, `font-bold`, `uppercase`, `letter-spacing-wide`, `color-neutral-900`; renders CMS `heading` field
- [ ] Body copy: `text-base`, `font-normal`, `color-neutral-700`; renders CMS `body` field; margin-bottom `24px`
- [ ] Unauthenticated state: two buttons in a row with `gap: 12px`:
  - "JOIN US" — primary button style (`color-primary` bg, white text, height `48px`, padding `0 24px`); routes to `/register`
  - "SIGN IN" — secondary button style (white bg, black text, `1px solid #111111` border, height `48px`); routes to `/login`
- [ ] Authenticated state: both buttons replaced by a single link "GO TO MY ACCOUNT →" (`text-base`, `font-medium`, `color-neutral-900`) routing to `/account`; heading and photograph remain unchanged
- [ ] Auth state read from a `useAuth()` hook (stub: `{ isAuthenticated: boolean }`); the hook implementation is out of scope — Dev provides a stub returning `false` by default
- [ ] All interactive element focus states match UX spec (border-focus outline)
- [ ] Section wrapped in `<section aria-label="Membership">`
- [ ] Responsive — mobile (sm): flex direction `column`; photograph moves below text; photograph `100%` width, `240px` height; buttons stack vertically full-width
- [ ] Responsive — tablet (md): flex direction `row` (same as desktop); photo width `320px`; horizontal padding `24px`
- [ ] `prefers-reduced-motion`: photo fade-in transition duration is 0ms

## Technical Notes
- `useAuth()` hook stub: create `src/api/hooks/useAuth.ts` with a single export `export function useAuth() { return { isAuthenticated: false } }` — a `// TODO(dev): replace with real auth` comment is required. The stub is enough for US-06 acceptance criteria.
- Button states (hover, focus, disabled) are implemented via Tailwind classes matching the UX spec component state tables — no inline styles
- The "→" in the authenticated link should be a Unicode right arrow (`→`) or an SVG icon — use Unicode to avoid icon dependency in this ticket
