# LP-013 — StorefrontLandingPage route assembly

## Type
[x] Feature  [ ] Chore  [ ] Bug  [ ] Migration

## Linked spec
docs/specs/storefront-landing-page.md — all user stories (full page assembly)

## Linked UX spec (if UI work)
docs/ux/storefront-landing-page.md — Screens / Views (document order of six sections), Accessibility (skip link, focus order, ARIA), Performance Notes

## Description
Assemble the `StorefrontLandingPage` page component by composing all section components in document order: Navbar, HeroBanner, NewArrivalsSection, ShopBySportSection, MembershipSection, Footer. Register the route at `/` in the React Router config. Add the skip-to-content link as the first DOM element. Ensure the React Query client and Zustand providers are correctly wrapping the tree.

## Acceptance Criteria
- [ ] `src/pages/StorefrontLandingPage.tsx` created as a default export (page-level component per SA conventions)
- [ ] Document order in JSX: skip link → `<Navbar>` → `<main id="main-content">` → `<HeroBanner>` → `<NewArrivalsSection>` → `<ShopBySportSection>` → `<MembershipSection>` → `</main>` → `<Footer>`
- [ ] Skip link is the first element: `<a href="#main-content" className="sr-only focus:not-sr-only ...">Skip to main content</a>` exactly as specified in UX accessibility section
- [ ] Route `/` registered in `src/router.tsx` (or equivalent) pointing to `StorefrontLandingPage`
- [ ] `<main id="main-content">` wrapper gives the skip link its target
- [ ] `document.title` set to `"Home | [Brand Name]"` on mount (via `useEffect` or a title hook)
- [ ] Page renders without layout shift: Navbar height offset `56px` is applied via `mt-[56px]` (or the token) on the `<main>` element so content does not hide under the fixed navbar
- [ ] All six sections are visible in the correct order when scrolling from top to bottom
- [ ] Browser routing works: clicking Navbar category links, hero CTA, product cards, sport tile SHOP buttons, and membership CTAs all navigate to the correct routes without full page reload
- [ ] `ReactQueryDevtools` is excluded from production builds (dev-only import)
- [ ] Lighthouse accessibility score ≥ 90 (verified by QA; Dev must ensure correct semantic structure)

## Technical Notes
- `StorefrontLandingPage` is the only default export in its file (rule from SA conventions)
- If `src/router.tsx` does not exist yet, create it with `createBrowserRouter` (React Router v6); mount `<RouterProvider>` in `src/main.tsx`
- `ReactQueryClientProvider` and Zustand (no provider needed — global store) should already be in `src/main.tsx`; if not, add them in this ticket
- Verify that the `max-content-width` (`1200px`) container is applied per section (each section manages its own container), not on the `<main>` element — the hero and footer are intentionally full-bleed
