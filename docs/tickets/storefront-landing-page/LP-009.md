# LP-009 — ProductCard component and New Arrivals section

## Type
[x] Feature  [ ] Chore  [ ] Bug  [ ] Migration

## Linked spec
docs/specs/storefront-landing-page.md — US-03 (New Arrivals Product Cards), US-04 (Wishlist on Product Card)

## Linked UX spec (if UI work)
docs/ux/storefront-landing-page.md — Section 3 — New Arrivals (layout, product card dimensions, colours, typography, ambiguity resolutions A-02/A-03/A-04, component states for card and heart icon, responsive behaviour)

## Description
Build two components: `ProductCard` (a self-contained card for a single product) and `NewArrivalsSection` (the grid wrapper that fetches and displays up to 3 new arrival products). `ProductCard` handles the sold-out state (UX A-02), the wishlist heart icon toggle (reading from and writing to `useWishlistStore` with LP-006 API mutation), and the card hover lift animation. `NewArrivalsSection` handles the "View All" link (UX A-03) and responsive grid layout.

## Acceptance Criteria
- [ ] `src/components/ProductCard.tsx` created as a named export; accepts `product: StorefrontProduct` prop
- [ ] Card layout: background `color-neutral-100`, border `1px solid color-neutral-300`, border-radius `0px`, shadow-sm
- [ ] Image container `240px` height (lg), `180px` (sm); `object-fit: contain`; image padding `16px`; `loading="lazy"`
- [ ] Text area: product name (`text-sm`, `font-medium`, `color-neutral-900`), subtitle (`text-sm`, `font-normal`, `color-neutral-600`), price (`text-base`, `font-medium`, `color-neutral-900`)
- [ ] When `product.price === null`: renders "Sold Out" text in `color-neutral-400`, `text-sm`, `font-normal` in the price slot
- [ ] Heart icon (`24×24px`) absolutely positioned `top: 12px`, `right: 12px`; always visible on all cards (including sold-out per UX A-04)
- [ ] Heart icon inactive: transparent fill, white stroke; active: `color-wishlist-active` (#FF0000) filled; transitions `transition-fast`
- [ ] Clicking the heart calls `useToggleWishlist` (LP-006); if unauthenticated, redirects to `/login`
- [ ] `<button aria-label="Add [product name] to wishlist">` with `aria-pressed="true/false"` on the heart button
- [ ] Clicking anywhere on the card (outside the heart button) navigates to `/products/:id`
- [ ] Card wrapped in `<article>`; inner `<a aria-label="[product name], [price or Sold Out]">` covers the full card area
- [ ] Card hover: `translateY(-2px)`, `shadow-md`, `transition-base`; disabled when `prefers-reduced-motion`
- [ ] Focus ring: `border-focus` (`2px solid #111111`) around entire card on keyboard focus

- [ ] `src/components/NewArrivalsSection.tsx` created as a named export
- [ ] Section heading "NEW ARRIVALS" — `text-xl`, `font-bold`, `uppercase`, `letter-spacing-wide`, `color-neutral-900`
- [ ] "View All" text link right-aligned to heading, `font-medium`, `text-sm`, `letter-spacing-wide`, underline on hover, routes to `/products?collection=new-arrivals`
- [ ] Product grid: 3-column CSS grid, `gap: 24px`; responsive: 2-column at md, 1-column at sm
- [ ] Fetches data via `useNewArrivals()`; shows skeleton cards during loading; shows error state on failure
- [ ] Section wrapped in `<section aria-label="New Arrivals">`
- [ ] Section padding: `48px` top/bottom, `40px` horizontal (lg); `24px` horizontal (md); `16px` horizontal (sm)
- [ ] Product images are lazy-loaded (`loading="lazy"`) per spec US-03

## Technical Notes
- `ProductCard` must be a pure presentational component: it receives `product` and an `onWishlistToggle` callback from `NewArrivalsSection` — do not couple the card directly to `useToggleWishlist`; pass the toggle as a prop for testability
- `NewArrivalsSection` owns the data fetching and wishlist toggle orchestration
- The unauthenticated redirect from the heart button must use the router's `navigate` function, not a hard reload
- Keep `ProductCard.tsx` under 300 lines; if it grows, extract `WishlistHeartButton` to its own file

## Out of Scope
- Product detail page
- "Add to cart" button on the product card (not in the landing page spec)
- Product filtering or sorting
- More than 3 product cards in this section
