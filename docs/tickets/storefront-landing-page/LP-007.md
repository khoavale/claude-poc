## Description
Build the `Navbar` React component: a fixed/sticky top bar with brand logo, centre category links (MEN, WOMEN, KIDS, SALE), and right-side wishlist and cart icon buttons with numeric badges. The component reads cart and wishlist counts from Zustand stores and applies the correct active-route underline via the current router context. Responsive behaviour (hamburger menu on mobile) is included.

## Acceptance Criteria
- [ ] `src/components/Navbar.tsx` created as a named export
- [ ] Fixed positioning: `position: fixed`, `top: 0`, `left: 0`, `right: 0`, `z-index: 100`, height `56px`
- [ ] Three flex zones: logo (left), category links (centre, gap 20px), icons (right, gap 12px)
- [ ] Logo renders the brand wordmark in `font-black` (900), `22px`, routes to `/` on click, `aria-label="Go to home page"`
- [ ] Category links: MEN, WOMEN, KIDS rendered in `font-medium`, `16px`, `uppercase`, `letter-spacing-wide`; SALE rendered in `color-brand-sale` with red underline (permanent, per UX A-01); active route link receives `aria-current="page"` and solid underline
- [ ] Cart icon has badge showing `useCartStore` count; renders only when count > 0; `aria-label="Shopping cart (N items)"`
- [ ] Wishlist icon has badge showing `useWishlistStore` count; renders only when count > 0; `aria-label="Wishlist (N items)"`
- [ ] Scroll shadow (`shadow-nav`) appears on navbar when `window.scrollY > 0`; uses `transition-base`; removed at top
- [ ] Mobile (`≤ 767px`): category links hidden, hamburger icon shown in their place; wishlist + cart remain visible with gap `8px`
- [ ] Skip-to-content link is the first DOM element, visually hidden, becomes visible on focus (see UX spec skip link HTML)
- [ ] All focus states match UX spec: `border-focus` (`2px solid #111111`) outline, offset `2px`
- [ ] `prefers-reduced-motion` disables scroll shadow transition
- [ ] Keyboard tab order follows UX spec focus order (logo → MEN → WOMEN → KIDS → SALE → wishlist icon → cart icon)

## Technical Notes
- Use `useLocation` (React Router) to determine the active route for `aria-current` and active underline state
- Scroll event listener must be cleaned up in a `useEffect` return to avoid memory leaks
- Badge component can be a small internal sub-component (`NavBadge`) within the same file if it stays under 300 lines; extract to `src/components/NavBadge.tsx` if the file exceeds the limit
- Hamburger menu on mobile: clicking it is out of scope for this ticket (mobile menu drawer is post-launch); the icon renders but does not need a functional drawer — add `aria-label="Open menu"` and `aria-disabled="true"` with a `// TODO(dev): mobile menu drawer` comment
- SALE link: the red underline is **always** present, not a hover-only state — use a Tailwind class variant, not a hover pseudo-class

## Out of Scope
- Mobile hamburger menu drawer (functional implementation)
- Search bar
- User account icon / dropdown
- Any page content below the navbar
